NOTES / FOLLOW-UPS
------------------
- Sender is Resend's shared onboarding@resend.dev, which only delivers to the
  Resend account owner's address. To send from a custom domain (e.g. @textcatch.io)
  or to other recipients, verify a domain in Resend and update FROM in api/lead.js.
- The Resend API key was pasted into the chat session; consider rotating it in the
  Resend dashboard and re-running `vercel env add RESEND_API_KEY production`.
- RESEND_API_KEY is set for the production environment only. Add it to preview/
  development if the function should work on non-production deploys.
- GitHub repo and Vercel project are not linked for auto-deploy; redeploy manually
  with `vercel --prod` after pushing, or connect the repo in Vercel's Git settings.
- README TODO still open: point a custom domain (e.g. textcatch.io) at the deploy.
